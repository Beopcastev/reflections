What happens when you initialize a repository? Why do you need to do it?

A: Initializes or creates a new Git repository. I need it to build my repository
   to have different versions.

How is the staging area different from the working directory and the repository? 
What value do you think it offers?

A: Staging area contains the files that are to be committed and working area 
   contains all the untracked files that we're working on. The commits are added 
   to repositories from staging area. It offers discretion to keep track of which
   file is where.

How can you use the staging area to make sure you have one commit per logical
change?

A: I run git diff, by this i can view all the changes in the staging area from the 
   working directory, and I can make sure there is only one logical change and then
   commit.

What are some situations when branches would be helpful in keeping your history 
organized? How would branches help?

A: When in situations, where the result isn't sure to be expected or to develop a 
   sub level,branches will be useful in having the history organized.

How do the diagrams help you visualize the branch structure?

A: They help to get a clear picture of reachability concept, to get a track of all
   the commits on the respective brances.
